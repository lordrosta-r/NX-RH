# DESIGN DATA EXPLOITATION & ROI - NANOXPLORE RH (10/10)

## OBJECTIF
Transformer les données brutes collectées via les formulaires d'évaluation en intelligence décisionnelle. Chaque Smart Block dans un formulaire n'est pas juste un champ : c'est une source de données structurées exploitables.

---

## 1. LES 5 DASHBOARDS STRATÉGIQUES

### Dashboard 1 : Flight Risk Radar (Risque de Départ)
**Audience** : RH, Manager principal.
**Source de données** : Smart Block "Météo" (satisfaction 1-10) + Smart Block "Mobilité" (flag oui/non + type) + tendance historique.
**Visualisation** : Scatter plot avec :
* Axe X = Satisfaction (1-10, tirée du dernier Smart Block Météo rempli).
* Axe Y = Performance (note globale de la dernière évaluation).
* Taille du point = Ancienneté.
* Couleur = Flag mobilité (rouge = veut partir, orange = mobilité interne, vert = stable).
**Action** : Cliquer sur un point → fiche employé. Le RH peut lancer une action de rétention.

### Dashboard 2 : Goal Gap Analysis (Objectifs : Fixés vs Atteints)
**Audience** : RH, Manager principal, Manager.
**Source de données** : Objectifs N-1 (target + réalisation), Smart Block "Import Dynamique N-1".
**Visualisation** : Bar chart empilé par département/équipe.
* Barre verte = Objectifs atteints.
* Barre jaune = Partiellement atteints.
* Barre rouge = Non atteints.
* Ligne de tendance comparative N-1 vs N-2 (progression d'année en année).
**Action** : Drill-down par équipe, puis par individu. Identifier les équipes sur-performantes et sous-performantes.

### Dashboard 3 : Skills Gap Map (Carte de Compétences)
**Audience** : RH, Manager principal.
**Source de données** : Smart Block "Compétences" (notation par compétence-clé), Aspirations (formations souhaitées).
**Visualisation** : Radar chart (spider chart) par département.
* Chaque branche = une compétence-clé (Leadership, Technique, Communication, Innovation…).
* Ligne bleue = Niveau moyen actuel de l'équipe.
* Ligne verte pointillée = Niveau cible défini par le RH.
* Zone rouge = Gap critique (delta > 2 points).
**Action** : Identifier les besoins de formation prioritaires. Exporter vers un plan de formation.

### Dashboard 4 : Sentiment Heatmap (Climat Social)
**Audience** : RH, Manager principal.
**Source de données** : Smart Block "Météo" agrégé par équipe, par département, par trimestre.
**Visualisation** : Heatmap calendaire (style GitHub contribution graph).
* Axe X = Mois/Trimestres.
* Axe Y = Équipes/Départements.
* Couleur = Score moyen (rouge = <4, orange = 4-6, vert = >6).
**Action** : Détecter les dégradations saisonnières (post-restructuration, post-évaluation…). Corrélation avec turnover.

### Dashboard 5 : Succession Pipeline (Viviers de Talents)
**Audience** : RH, Manager principal.
**Source de données** : Performance (note globale), Ancienneté, Aspirations ("évolution souhaitée"), Flag mobilité ("veut évoluer en interne").
**Visualisation** : 9-Box Grid classique (Performance x Potentiel).
* Axe X = Performance (tirée de la dernière évaluation).
* Axe Y = Potentiel (estimé via : aspiration d'évolution + note croissante sur 3 ans + formation demandée).
* Chaque case = nombre d'employés + noms.
* Case top-right = "Stars" (haute performance + haut potentiel) → Successeurs identifiés.
**Action** : Le manager principal clique sur une case → voit la liste. Il peut taguer un employé comme "Successeur potentiel pour le poste X".

## 2. TABLEAU DES SOURCES DE DONNÉES

| Smart Block / Source | Type de Donnée | Dashboards Alimentés |
|---|---|---|
| Météo (Satisfaction 1-10) | Numérique | Flight Risk, Sentiment Heatmap |
| Mobilité (Flag + type) | Booléen + Enum | Flight Risk, Succession |
| Import Dynamique N-1 | Objectifs structurés | Goal Gap Analysis |
| Note Globale | Numérique | Flight Risk, Goal Gap, Succession, Calibration |
| Compétences (si Smart Block activé) | Radar multi-dimension | Skills Gap Map |
| Aspirations (texte) | Texte + Flags | Succession, Skills Gap |
| Demande salariale | Numérique | Inbox RH (pas dashboard, traitement manuel) |

## 3. ROI / GAINS ESTIMÉS (Argumentaire Métier)

### Valeur Business de chaque Dashboard

| Dashboard | Problème résolu | Gain estimé |
|---|---|---|
| Flight Risk Radar | Un départ non anticipé coûte 6-12 mois de salaire en recrutement + formation | Réduction de 15-25% du turnover non souhaité |
| Goal Gap Analysis | 40% des objectifs ne sont jamais relus → désalignement stratégique | +20% d'alignement objectifs/stratégie |
| Skills Gap Map | Les plans de formation sont déconnectés des besoins réels | -30% de budget formation gaspillé |
| Sentiment Heatmap | Le climat social se dégrade silencieusement → départs en série | Détection précoce : -3 mois de réaction |
| Succession Pipeline | Les postes-clés vacants restent vides 4-8 mois | -50% de temps de remplacement |

### Coût de l'Inaction vs. Coût de l'Implémentation

| Métrique | Sans NX-RH Data | Avec NX-RH Data |
|---|---|---|
| Temps pour identifier un risque de départ | 3-6 mois (quand la lettre arrive) | Temps réel (dès la Météo < 4) |
| Précision du plan de formation | ~30% (basé sur intuition) | ~75% (basé sur données Skills Gap) |
| Taux de succession interne | ~20% (pas de vivier) | ~50% (pipeline identifié) |
| Temps RH pour consolider les données | 2-3 semaines/campagne (Excel) | Automatique (Smart Blocks structurés) |

## 4. ARCHITECTURE DATA

### Accès aux Dashboards

Les dashboards sont accessibles depuis **deux points d'entrée** :

* `/hr/analytics` → RH voit **tous** les dashboards sur **toute l'organisation**.
* `/manager` → Un manager avec visibilité étendue voit les mêmes dashboards, filtrés sur sa branche uniquement.

Le Manager n'a **pas** accès aux dashboards analytics (il voit uniquement les données de son équipe directe dans son Dashboard classique).

### Filtres Communs à Tous les Dashboards

* Période (campagne N, N-1, N-2… ou plage libre).
* Département / Service.
* Niveau hiérarchique.
* Manager.
* Ancienneté (tranches).

### Export

* PDF (rapport visuel avec graphiques).
* CSV (données brutes pour traitement externe).
* Planifié (envoi automatique d'un rapport mensuel/trimestriel par email au RH).

---

## 5. CONSIDÉRATIONS RGPD

* Les dashboards affichent des **données agrégées** par défaut. Le drill-down vers l'individu n'est possible que pour le RH et le manager responsable de la branche.
* Les données de type "Météo" et "Aspirations" ne sont **jamais** exposées nominativement dans les exports CSV — elles sont anonymisées par défaut.
* Le Smart Block "Demande Salariale" n'alimente **aucun** dashboard. Il est traité manuellement dans `/hr/requests` par le RH uniquement.
* Durée de rétention configurable par l'Admin dans `/admin/compliance` (par défaut : 5 ans après la dernière campagne de l'employé).
