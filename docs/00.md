# NanoXplore RH — Essence du produit

## Qu'est-ce que NanoXplore RH ?

NanoXplore RH est une **plateforme web d'entretiens annuels self-hostée**, conçue pour les organisations qui veulent maîtriser intégralement leur processus de gestion de la performance — sans dépendre d'un SaaS externe, sans export de données RH vers des tiers.

Elle couvre l'intégralité du cycle : de la création d'une campagne d'évaluation jusqu'à la validation finale de l'entretien par le manager, en passant par l'auto-évaluation de l'employé, la revue des objectifs passés et la définition des objectifs futurs.

**Pour qui ?**
- Les équipes RH qui pilotent le processus
- Les managers qui conduisent les entretiens
- Les employés qui participent activement à leur évaluation
- Les administrateurs système qui maintiennent la plateforme

---

## Le cycle annuel en 6 phases

Chaque campagne d'évaluation se déroule en 6 phases chronologiques et ordonnées. Une phase ne peut pas être complétée avant la précédente.

```
┌──────────────────────────────────────────────────────────────────┐
│  Phase 1 — PRÉPARATION                                           │
│  Qui : RH                                                        │
│  Quoi : Créer la campagne, sélectionner les templates de         │
│         formulaires, définir les dates, ouvrir la campagne       │
├──────────────────────────────────────────────────────────────────┤
│  Phase 2 — AUTO-ÉVALUATION                                       │
│  Qui : Employé                                                   │
│  Quoi : Remplir son bilan personnel — forces, axes              │
│         d'amélioration, ressenti sur l'année écoulée             │
├──────────────────────────────────────────────────────────────────┤
│  Phase 3 — BILAN N-1                                             │
│  Qui : Employé + Manager                                         │
│  Quoi : Révision des objectifs de la période précédente —        │
│         atteints, partiellement atteints, non atteints           │
├──────────────────────────────────────────────────────────────────┤
│  Phase 4 — OBJECTIFS FUTURS                                      │
│  Qui : Employé + Manager                                         │
│  Quoi : Définir les objectifs mesurables pour la prochaine       │
│         période (SMART ou libres selon le template)              │
├──────────────────────────────────────────────────────────────────┤
│  Phase 5 — ASPIRATIONS                                           │
│  Qui : Employé                                                   │
│  Quoi : Développement professionnel souhaité, mobilité           │
│         interne, formations, aspirations de carrière             │
├──────────────────────────────────────────────────────────────────┤
│  Phase 6 — ENTRETIEN                                             │
│  Qui : Manager (+ Employé en présence)                           │
│  Quoi : Conduire l'entretien, noter, commenter chaque section,   │
│         valider l'évaluation finale                              │
└──────────────────────────────────────────────────────────────────┘
```

## Les 4 rôles en un mot

| Rôle | Résumé | Position dans l'org |
|---|---|---|
| **Admin** | Configure et maintient le système | Transverse — technique |
| **HR** | Pilote le processus RH, crée les templates et les campagnes | Transverse — fonctionnel |
| **Manager** | Manage son équipe directe, conduit les entretiens | Hiérarchique N+1 |
| **Employee** | Participe à son évaluation, remplit ses formulaires | Base |

---

## Concept clé : templates avant campagne

HR crée d'abord des **templates de formulaires** (modèles réutilisables). Ce n'est qu'une fois qu'au moins un template est disponible qu'une campagne peut être lancée. Les managers ne créent pas de formulaires — ils choisissent parmi les templates fournis par HR.

Cette règle garantit :
- L'uniformité des évaluations dans l'organisation
- La maîtrise du processus par les RH
- La réduction de la charge pour les managers

---

## Ce que NanoXplore RH ne fait PAS (scope out)

- **Pas de paie** — aucune donnée salariale, aucun lien avec les systèmes de paie
- **Pas de recrutement** — pas d'ATS, pas de gestion de candidatures
- **Pas de formation** (v1) — les aspirations sont enregistrées mais pas traitées automatiquement
- **Pas de planning / congés** — aucune gestion des absences
- **Pas de messagerie interne** — les notifications se font par email sortant uniquement
- **Pas de SaaS multi-tenant** — une instance = une organisation, self-hosted
- **Pas de mobile native** — l'interface est responsive mais il n'y a pas d'app iOS/Android
